# NLDI API Endpoint Outline

Base path: `{prefix}/` (configured via server config)

## Root

| Path | Description | Query Params |
|------|-------------|--------------|
| `/` | Landing page — title, description, links | |
| `/about/health` | Health check (server, db, pygeoapi) | |
| `/openapi` | Redirect → `/docs` | |
| `/docs` | Swagger UI (auto-generated by Litestar) | |

## Linked Data (`/linked-data`)

| Path | Description | Query Params |
|------|-------------|--------------|
| `/` | List all data sources | |
| `/hydrolocation` | Hydrologic location nearest to coordinates | `coords` (WKT) |
| `/comid/position` | Find flowline by spatial point lookup | `coords` (WKT) |
| `/{source_name}` | List all features for a source | `f`, `limit`, `offset` |
| `/{source_name}/{identifier}` | Get single feature by source + ID | `f` |
| `/{source_name}/{identifier}/basin` | Upstream basin polygon | `simplified`, `splitCatchment` |
| `/{source_name}/{identifier}/navigation` | Navigation mode URLs | `f` |
| `/{source_name}/{identifier}/navigation/{nav_mode}` | Data sources available for a nav mode | |
| `/{source_name}/{identifier}/navigation/{nav_mode}/flowlines` | Navigate flowlines | `distance`, `trimStart`, `excludeGeometry` |
| `/{source_name}/{identifier}/navigation/{nav_mode}/{data_source}` | Navigate features of a data source | `f`, `distance`, `excludeGeometry` |

### Path Parameters

- `source_name` — crawler source identifier (e.g. `wqp`, `nwissite`); `comid` is always valid
- `identifier` — feature ID within the source
- `nav_mode` — navigation mode: `UM`, `UT`, `DM`, `DD`
- `data_source` — target data source for navigated features

### Common Query Parameters

- `f` — output format: `json` (default), `jsonld`, `html`
- `distance` — navigation distance in km (defaults vary by nav mode)
- `coords` — WKT point geometry, e.g. `POINT(-89.509 43.087)`
